Overview

This tool is provided to help designers configure the microcontroller I/Os. You can choose the most suitable microcontroller for your application by using the Tools->Multiple MCUs Scanning tab and specifying the required peripherals. You can either:

assign the known required signals (functions) to pins (within limits; the tool applies certain rules to these assignments, and logical blocks of signals are assigned together), or

select the peripheral modes required and let MicroXplorer assign the pins. If the pin location is important, select the Keep User Placement checkbox.

         Installation. Double click on MicroXplorer-Setup.exe file, and browse to choose the installation path (e.g. Extract to : C:\Program Files\MicroXplorer_V1.1\). Then create a shortcut to C:\Program Files\MicroXplorer_V1.1\MicroXplorer.exe on your desktop.

         Run. From your desktop, double-click on MicroXplorer.exe shortcut.

Tutorial

1.      Tutorial Example

In the tutorial, you will find and configure an STM32 Microcontroller (MCU), in order to use the Ethernet and USB peripherals.

 

2.      Aim of the tutorial

         To find a STM32 MCU that contains at least one Ethernet, one USB-OTG, and one ADC.

         To optimize the pin mapping of the chip in order to use the maximum amount of peripherals with the smallest package (minimum number of pins).

         To know which programming modes are available for the Ethernet and USB-OTG peripherals.

         To map certain Peripheral Modes by letting the MicroXplorer assign the pins to the signals (i.e. functions) required for the USART3 and USB-OTG FS-Dual-Role-Device, and ADC1-IN4 modes.

         To Map Ethernet MII, even if no pins are left in the current configuration, by letting the MicroXplorer remap all the selected peripheral modes (No Keep User Placement).

         To physically map an IO to the PD8 pin (Chip View).

         To manually force the remapping of Ethernet MII to another pin.

         To save your work.

         To reload an older configuration

Chip View

The Chip View in the right-hand window shows a graphical representation of the pinout of the selected MCU in the selected package (BGA, QFP,...). Each pin is represented with its name (e.g. PC4) and the current function assignment (e.g. ETH_MII_RXD0).

To use Chip View requires you to have knowledge of the MCU and its pin assignment. In this view, individual pins can have specific functions assigned to them.

Some basic controls are built-in (see Rules section) for example to ensure the coherency of blocks of pins.

To modify a pin assignment you have to click on the pin. A list of possible assignments is shown (other possible alternate functions) with the current assignment highlighted. A click on the new function modifies the pin assignment.

You can Remap a function to another pin, by hitting the CTRL key and moving it to the target pin.

Caution: Pin assignment performed from Chip View is destructive, i.e. the previous assignment will be lost.

Peripherals panel

You can simply use the left hand window (i.e. the Peripherals Panel) to select the mode of a peripheral (e.g. Full-Duplex-Master mode for the SPI1). Each time a mode is set, the Chip View in the right window displays the new state of the pin(s).

If the selected peripherals use the same pins, MicroXplorer will try to resolve the conflicts by remapping the functions pins for this mode. See Rule4 for an example.

In case the conflict cannot be resolved by remapping only this mode, you can

         either try to select the peripherals in a different sequence ( Keep User Placement checkbox is selected),

         or let MicroXplorer try all the remap combinations for all the blocks to find a solution (Keep User Placement checkbox is Not selected).

If the conflict remains, MicroXplorer displays a Conflict message on the affected peripheral.

Note: Tree Interface or Accordion Interface are two ways of displaying Peripheral panel.

Glossary

Block

A "block of pins" is a group of pins that can all be assigned together, to make a peripheral mode. For instance PC1, PA1, PA2, PA7, PC4, PC5, PB11, PB12, PB13, PB5 for RMII mode of Ethernet Peripheral. Block coherence is needed for STM32F100x/ F101x/ F102x/ F103x and STM32F105x/ F107x.

Default block

Default pin/function assignments for a given peripheral mode. For instance USART3 SYNCHRONOUS mode default assignments are:

  • PB10.USART3_TX
  • PB11.USART3_RX
  • PB12.USART3_CK

Remap block

Remapped pin/function assignments for a given peripheral mode. For instance USART3 SYNCHRONOUS mode remap assignments are:

  • PB10.USART3_TX
  • PD9.USART3_RX
  • PD10.USART3_CK

Keep User Placement

         When the Keep User Placement checkbox is selected, each of the functions for the chosen peripheral mode stay allocated (mapped) on a given pin. In this case, MicroXplorer cannot change the function location from one pin to another.

If a function becomes unavailable (green pin), you should find out which other function(s) use these pins and deselect them one by one to determine which one is in conflict. Once done, you can reselect the functions in the new sequence and they should find other remap pins. See Rule4 for an example. This can be a time consuming process and it is normally better to unselect the Keep User Placement checkbox.

Keeping User Placement can be useful in cases where you have mapped a function to one pin from the Chip View, and you need it to stay fixed there.

You may also want to lock the pins of all the peripherals that you have already configured using the Peripherals panel.

Keep User Placement can be selected/unselected at any time during the configuration..

 

No Keep User Placement

After unselecting the Keep User Placement checkbox, MicroXplorer will find a solution to your next request by remapping some of the previously mapped blocks. In this case, MicroXplorer is allowed to move previously mapped functions to other pins.

In other words, your next peripheral or function choice may cause some of your previous functions to be remapped.

Not Keeping User Placement is useful when you want to optimize the placement of your peripheral, i.e. if you want to get the maximum number of peripherals on the smallest chip.

Keep User Placement can be selected/unselected at any time during the configuration.

 

Manual Remapping

You can manually remap a function to another pin from the Chip View by hitting the CTRL key and clicking at the same time on the source. The target pins will then appear in a blue color, and you will be able to move your function by dragging the mouse pointer from the source pin to the target pin.

 

Manual Remapping with destination pin ambiguity

For MCUs with block coherence (STM32F100x/ F101x/ F102x/ F103x and STM32F105x/ F107x), the destination pin can be ambiguous, i.e. we can have more than one destination block including the destination pin. In that case, just move the mouse pointer slowly over the target pin so MicroXplorer shows you each possible alternative remapping block.

 

Conflict colors and messages

         Peripheral color

The peripheral color can be:

o   Black: peripheral is not configured (no mode is set), and all configurations (modes) are available.

o   Green: peripheral is configured (one mode is set), and all other modes are available (see below).

o   Green with orange warning: peripheral is configured (one mode is set), and not all other modes are available (see below).

 

o   Orange: peripheral is not configured (no mode is set), and not all modes are available (see below).

o   Red: peripheral is not configured (no mode is set), and no mode is available (see below).

         Mode color

The peripherals mode color can be:

o   No color: mode is available (see below).

o   Orange: mode is partially available (see below).

o   Red: mode is not available (see below)

 

 

         Peripheral pop-up Partly disabled warning

A pop-up message is displayed above the peripheral menu, if not one of its modes is available.

E.g. the I2C2 peripheral is not fully available because some of its modes are not available. The message says that some of its pins are allocated to ADC1 IN4 signal (see below).

         Peripheral pop-up Not Available warning

A pop-up message is displayed above the peripheral menu, if none of its modes are available.

E.g. the I2C2 peripheral is not available because for each mode, at least one of its functions has no (re)mapping available. The message says that some of its pins are allocated to USART3 and/or to PB10 I/O (see below).

 

         Mode pop-up warning

A pop-up message is displayed above the peripheral mode item, if at least one of the mode functions is not available, i.e. if all its (remap) pins are already allocated (see below).

F.A.Q.

1.      Why does MicroXplorer move some functions when I add a new peripheral mode?

You may have unselected the Keep User Placement checkbox. In this case, MicroXplorer performs automatic remapping to optimize your placement.

2.      How can I manually force the remapping of a function?

You should use the Manual Remapping feature.

3.      Why some pins are colored in yellow in Chip View?

They are Power pins, and they are not available for peripheral signals.

4.      Why cant I change the function of some pins? (When I click some pins nothing happens)?

Some special pins are not available for peripheral signals, like Power pins (yellow), or Boot pins (light green)

5.      How can I use MicroXPlorer as an Eclipse plug-in?

Refer to Install MicroXplorer Eclipse Plugin documentation available on the ST Internet site.

Annexe1 - Rules

RULE 1: Block coherency

All pins in a block are mapped/unmapped at the same time, (if there is no ambiguity about the mode to be mapped/unmapped).

Note: There is Ambiguity about a block of pins in case there are several possible peripheral modes that can need this block.

  • e.g. Case of RMII group not mapped on PA7, PC4, PC5,... :
    If you set the ETH_RMII_RXD0 function on PC4, then MicroXplorer sets the rest of the Ethernet block: CRS_DV, RXD1,... on PA7,PC5,...(Figure 1)
  • e.g. Case of ETH_RMII group already mapped on PA7, PC, PC5,... :
    If you set ADC1-IN15 on PC5, then MicroXplorer unsets the Ethernet block, and then you must set it on PD8 (Figure 2), ... (Rule4).

Figure 1: Rule 1

 

 

^top

 

RULE 2: Block inter-dependency

Two blocks cannot be selected at the same time (alternate blocks are cleared).

e.g. If the RMII_RXD0 function is set on PD9 and the user then sets it on PC4, MicroXplorer will remove the RMII_RXD0 function from PD9 (MicroXplorer also moves any other assigned pins in the block. Rule4).

 

^top

RULE 3: One block = one peripheral mode

When a pin block is assigned in Chip View the related peripheral mode is set in the Peripherals Panel (left window).

e.g. After assigning the RMII_RXD0 function on PC4, RMII Ethernet mode is automatically selected in the left-hand window.

 

^top

RULE 4: Block remapping (only for STM32F10x)

When trying to set a peripheral mode, if at least one pin in the default block is already in use, MicroXplorer try to find a remap block.

  • E.g. MicroXplorer remaps USART3s Hardware-flow-control mode to the (PD8-PD9-PD11-PD12) block, because PB14, (part of its default block) is already allocated to the SPI2_MISO function,...(Figure 2)

Figure 2: Rule 4

 

If MicroXplorer cannot find a remap block, either you must select the functions in a different sequence, or you must unset the Keep User Placement checkbox, and let MicroXplorer remap all the blocks to find a solution.

Note For STM32F10x, all the functions of a block must be mapped to a pin with the same remap number.

 

^top

RULE 4: Remapping the functions of a block (only for STM32F20x)

When trying to set a peripheral mode, if one pin of the default block is already in use, MicroXplorer tries to find a remap pin.

Note For STM32F20x, each all the functions in a block can be mapped separately to a pin with a different remap number.

 

^top

RULE 5: Block shifting (only No Keep User Placement )

If a block cannot be mapped, and there are no free alternate ones, MicroXplorer tries to remap all the peripheral modes impacted by the shared pin.

  • E.g. If you first set USART3 Asynchronous mode then Asynchronous (PB10-PB11) default block is set, and ETH_MII becomes unavailable (red)... (Figure 3).

 

Figure 3: Rule 5

 

 

  • Then, if the you uncheck the Keep User Placement checkbox you are then allowed to set ETH_MII mode, but USART3 Asynchronous mode has been shifted to PC10-PC11 block by MicroXplorer (Figure 4),

 

Figure 4: Rule 5

 

 

^top

RULE 6: Setting / Clearing a peripheral mode

There is a link between the Peripherals panel and Chip View.

When a peripheral mode is set or cleared, all the pin functions of the block are set or cleared.

 

^top

 

RULE 7: Mapping a function individually (Keep User Placement)

What happens to a Function that has been set individually (no peripheral mode is set) if MicroXplorer needs its pin?

         If Keep User Placement has been set: MicroXplorer cannot change the pin location of the function, i.e. it cannot shift this function to another pin.

         If Keep User Placement has NOT been set: MicroXplorer can change the pin location of the function.

 

RULE 8: One IO is mapped

e.g. Input_Floating is mapped on PC5.

MicroXplorer cannot change the pin location of the IO, i.e. MicroXplorer cannot shift this IO to another pin.
